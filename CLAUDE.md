# Project Guidelines

## Documentation

Wiki articles for this project use a **design/specification lens**, not a coding lens.

**Include:** intent, behaviour contracts, decisions and rationale, trade-offs, operational model, user journey, limitations, risks, comparisons with related tools.

**Exclude:** code snippets, specific command syntax, implementation mechanics, bash-version workarounds. These belong in the source code.

Before writing or reviewing a wiki article, ask for each element: "Does a reader need this to understand the system's intent and behaviour, or to implement it?" If the latter, exclude it.
